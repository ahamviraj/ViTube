# Contributing to ViTube

First of all, thank you for considering contributing to ViTube.

We welcome bug reports, feature requests, documentation improvements, and code contributions.

---

## Development Guidelines

* Follow Kotlin coding conventions.
* Keep functions small and focused.
* Prefer readability over clever code.
* Use Jetpack Compose best practices.
* Keep UI and business logic separate.
* Avoid deprecated Android APIs.

---

## Pull Requests

Before opening a pull request:

* Ensure the project builds successfully.
* Test your changes.
* Keep commits meaningful.
* Write clear commit messages.

Example:

```
Add media scanner
```

instead of

```
Update files
```

---

## Issues

When reporting a bug, please include:

* Android version
* Device model
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots if applicable

---

## Code Style

* Kotlin only
* MVVM architecture
* Material 3
* Coroutines
* Flow
* Clean Architecture principles

---

## Thank You

Every contribution, whether it's fixing a typo or implementing a feature, helps make ViTube better.
